## 1. 一句话与愿景

四名玩家在**鉴宝 / 摸金 / 集装箱遗物**题材下，围绕「格子里的未知宝箱」做**信息不完全**的公开竞价；每轮根据公共与私密情报刷新**价格区间**，用**固定档位加价**（非键盘输数字）出价，三轮后结算——**上手极简单**（甚至可「看区间乱出价」也有赢面），高手通过**信息差与读心**获利。

---

## 2. 设计原则（来自方案思考逻辑）

1. **易上手**：不强依赖心算；系统给出**钱数范围/区间**，降低认知负担。  
2. **禁止「暗拍主导」**：首版明确 **不做暗标作为主机制**；信息需可置换、可讨论空间（虽当前局内讨论弱，但设计上保留信息渐进公开，而非一次性密封）。  
3. **不做「速胜碾压」**：不设「差距过大即直接判赢」，避免新手场被无限秒杀、无基础体验。  
4. **首版不做角色技能**：技能与复杂随机后置，避免「一用技能就明牌」与平衡爆炸。  
5. **出价即信息**：与德州不同——下注与信息在 BidKing 中耦合，**会玩的人在前几回合会持续暴露内心底价区间**（见第 8 节）。  
6. **互动呈现**：局内必须出现**四人头像**、出价**气泡**；可用 **Live2D 拍卖员** 做规则讲解与氛围，**控制静态 UI 量**。  
7. **强制结构**：一局 **3 轮**固定；时间与最高价**同步公示**（见流程）。

---

## 3. 与德州等品类的差异（数值文档核心）

| 维度 | 德州等 | 竞拍之王 |
|------|--------|-----------|
| 私有信息 | 手牌长期私有 | 部分「私密信息」会通过系统机制变成**他人可见的系统信息** |
| 下注与信息 | 相对分离 | **出价本身是信息的一部分**，携带信息熵 |
| 完全信息风险 | — | 若所有人极度理性且信息过满，会趋近**完全信息公开竞价**，趣味性下降 → 需控制 **I 的强度**并引入局外变量 **M**（任务、套装等，可后置） |

**玩家心理模型（抽象）**

- \(E[V|I]\)：基于当前及历史信息 \(I\) 对真实价值 \(V\) 的估计。  
- \(B\)：本回合出价。  
- \(C\)：本回合为获得信息支付的额外成本（若有道具/情报消费）。  
- 扩展：\(M\) 为场外变量（任务、套装组件等），使估价公式不单由局内 \(I\) 决定。

## 4. 核心循环与一局结构

### 4.1 宏观三阶段（方案原文）

| 阶段 | 目的 |
|------|------|
| **阶段一** | 公布价格与**最少信息** |
| **阶段二** | **继续差异化信息**（公共 + 个人私密渐进） |
| **阶段三** | **公开竞价**（限时加价，规则见下） |

### 4.2 参考品类与底层玩法

- 参考超休闲 **Bid Wars 1** 式体验：有**底价/起拍**，众人在此基础上**浮动加减**。  
- 内核描述：**数格子 + 信息差异化博弈**（背包格子与轮廓见第 5 节）。

### 4.3 单局阶段流程（方案细化）

以下模块在 PRD 中记需求级描述；**主页、选场次**本期按下不表。

1. **公布阶段**  
   - 收**门票**（经济规则待运营定档）。  
   - **NPC 公布起拍价/底价**，划定**初始价格区间**，避免完全看不懂。

2. **轮次 1、2**（结构相同）  
   - 公布**公共信息**。  
   - 公布**你的私密信息**（每人不同）。  
   - 综合信息后，**系统刷新价格区间**（不向玩家要求手算）。  
   - **倒计时开始**，全员用下方**固定档位加价**（见 6.2）。  
   - **倒计时进行到一半**时，**公布场上当前最高价**（照顾新手「跟大概」）。  
   - **结算本轮**：按规则决出最高出价者；**出价最高者获得额外信息**（为下轮信息差服务）。

3. **轮次 3**  
   - 同样：公共信息 + 私密信息 → 区间更新 → 倒计时加价。  
   - 差异：**若倒计时结束仍无人出价，则直接走过并结算**（避免拖死）。

4. **局末结算**  
   - 表现：**流光**、加速字符结算、**进度条充能**等。  
   - 若有配置：**随机事件**在此时机触发。  
   - **结算货币**；**拍得者选择是否卖出**（二次决策，具体数值与 UI 待补）。  
   - 结算结束。

## 5. 背包、格子与物品

- **格子**：倾向 **5×7 = 35 格**，单格 **1×1**；物品占位含 **1×4、2×2、4×4、1×3、2×3** 等（与方案一致）。  
- **单局物品数量**：约 **7～15 个**「有价格与轮廓」的单位（与轮廓种类对应）。  
- **品质色阶（缩减后）**：**白 / 蓝 / 紫 / 金 / 红**。  
  - **红色**：**命运向物品**，可触发**估值反转**等强戏剧效果；可支持「请人鉴定」等玩法钩子（高端场）。  
  - **套装**：多件组成一套可**翻价/加成**（偏高端维度）。  
- **展馆等功能**：明确 **后置**；早期可用「大虚数投放点戒指/礼物卡/限量」等运营占位，不挡 MVP。

---

## 6. 出价与 UI/UX

### 6.1 禁止主流程「键盘输数字」

- 主出价方式为 **少量按钮 + 固定倍率档位**（例：类似斗地主的加倍感，如 **100 / 1000 / 10000** 量级，具体数值策划定）。  
- 避免自由输入：操作慢、易错、且不利于「出价即信号」的设计。

### 6.2 信息展示

- **上方**：**问号（未知）**与当前能给出的**钱数范围/区间**（随情报更新）。  
- **场景化**：宝箱被掩埋、仅露出轮廓与色块等，强化摸金/鉴宝感知（与宣发一致）。

### 6.3 社交与互动（当前版）

- **2v2 组队**：**后续**再说。  
- 当前强调：**四人可见**、**出价气泡**、**Live2D 拍卖员**（讲解/氛围）。

---

## 7. 胜利条件与经济（PRD 级）

- **单局胜利定义**：三轮竞价与结算规则下，**以最终货币/资产变化或「捡漏/误判」体验目标**为主（方案侧重过程与情感曲线，非单一「淘汰三人」）。  
- **不设**：「领先超过某阈值即立刻获胜」。  
- **门票 + 卖出选择**：计入局内经济闭环；具体数值表见第 9 节与附录。

---

## 8. 数值与分层体验（整合数值文档）

### 8.1 出价与估价误差 \(\sigma\)

- 在零额外花费的简化模型下，用系数 **k∈(0,1)** 描述情报质量：情报越准，**\(\sigma\) 越小**；竞价越透明、对手信息越多，**k 往往越大**（文档原意：需在设计中平衡）。  
- **经验区间**：高手 \(\sigma\) 可 **小于 5%**；萌新 **约 30%～50%**。  
- **设计目标**：高手赚的是与萌新之间的**信息差钱**；若全员 \(\sigma\) 过小 → 接近完全信息竞价 → 趣味性下降。

### 8.2 新手局 vs 高手局（信息权重示意）

文档用 **w1、w2、w3** 表示不同信息来源权重（具体对应字段实现时命名即可）：

| 场次倾向 | w1 | w2 | w3 |
|----------|----|----|-----|
| **新手局** | 高 | 中 | 低 |
| **高手局** | 低 | 高 | 中 |

含义归纳：**新手局**靠大量系统提示把众人 \(\sigma\) 尽量压到 **约 20% 以内**，在小范围博弈里找爽感；**高手局**通过信息交叉与读心，使 public+private 信息对真相的覆盖**约 ≤70%**，留 **约 30%** 推理与博弈空间。

### 8.3 单局结果类型与目标占比（监测用）

| 类型 | 利润率（文档示例） | 设计备注 |
|------|-------------------|----------|
| 正常价买下 | 约 -15%～-30% | 常态 |
| 捡漏型 | 大于 30% | 爽点 |
| 误判型 | 约 -40%～15% | 过多会让人觉得信息不可靠 |
| 被骗型 | 小于 -40% | 局内可有「骗人者爽、被骗者仍可接受」的调性目标 |

**文档建议监测比例（示例）**：正常约 **55%**，捡漏约 **25%**，误判约 **15%**，严重被骗约 **5%**（上线后按数据迭代）。

### 8.4 场次与「区间相对真实价值」倍率

用户看到的**上下限区间**相对「集装箱/物品」真实价值的倍率范围（策划可调）：

| 场次 | 区间倍率（相对真实价值） |
|------|---------------------------|
| 新手场 | **0.5x ~ 2x** |
| 中级场 | **0.2x ~ 5x** |
| 高级场 | **0.08x ~ 12.5x** |
| 娱乐场 | **0.01x ~ 100x** |

**同品质内方差**：品质之间拉距大；同品质内方差策略与**红色命运物的大反转**配合，避免「全明牌后只剩算计」或「全随机无学习」。

## 9. 题材与宣发方向（方案）

- **蹭量与风格**：整体类似「**三角洲行动**遗物拍卖」的**紧张战利品感**，但题材上走 **泛鉴宝 + 摸金 + 超自然/宝箱** 等更易传播的组合。  
- **视觉**：宝箱被掩埋、不同**轮廓与颜色**外露，与格子占用一致。

## 11. 术语表

| 术语 | 定义 |
|------|------|
| 区间 / 范围 | 系统根据情报给出的可理解价格带，降低心算 |
| 固定档位加价 | 预设金额按钮加价，非自由数字输入 |
| 中场最高价 | 倒计时过半时公开的当前最高出价 |
| 红色命运物 | 红色品质，可承担强反转、鉴定等戏剧机制 |
| 信息熵 / 出价即信息 | 出价行为向他人泄露你对 \(V\) 的判断区间 |
| \(\sigma\) | 估价误差；用于分层与难度调节的抽象指标 |

---

## 12. 待补全项（缺口清单）

1. **门票定价、货币命名、局外成长**是否与「卖出」二次结算联动。  
2. **固定档位**具体金额与局内通货膨胀曲线。  
3. **轮次 3「无人出价」**时的物品归属与价格认定规则（流拍是否可能）。  
4. **「出价最高者得额外信息」**的具体信息类型与防滚雪球边界。  
5. **Live2D** 资源范围与是否首包内置。  
6. **w1/w2/w3** 与系统字段的一一映射表（程序用）。  
7. Word 内嵌「电子表格」图片中的完整数值表，需策划导出后并入附录。
